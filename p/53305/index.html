Reading Note - 《白帽子讲 Web 安全》

  • 第一篇 世界观安全

    • 第一章 我的安全世界观
      • 安全的本质
        • 信任域：通过安全检查（过滤、净化）的过程划分出来的具有不同信任级别的区域
        • 信任边界：划分两个不同信任域之间的边界
        • 安全问题的本质是信任的问题
      • 安全是一个持续的过程
        • 微软推崇的安全开发流程
          安全开发流程（SDL）将安全检查贯穿于整个软件生命周期中。而安全检查本身也需要不断更新，增加针对新型攻击方式的检测与防御方案。
      • 安全三要素
        • 机密性：要求保护数据内容不能泄露，加密是实现机密性要求的常见手段
        • 完整性：要求保护数据内容是完整、没有被篡改的。常见的保证一致性的技术手段是数字签名
        • 可用性：要求保护资源是“随需而得”，如拒绝服务攻击（Dos）破坏的是安全的可用性
      • 安全评估
        • 1 资产等级划分
          • 互联网公司的核心资产：数据
        • 2 威胁分析
          • 威胁：可能造成危害的来源
          • 威胁建模：STRIDE 模型
            伪装、篡改、抵赖、信息泄露、拒绝服务、提升权限
          • 头脑风暴，确定攻击面
        • 3 风险分析
          • 风险：可能会出现的损失
          • DREAD风险模型
            Damage Potential、Reproducibility、Exploitability、Affected users、Discoverability
        • 4 设计安全方案
          能够有效解决问题；用户体验好；高性能；低耦合；易于扩展与升级
      • 安全原则/指导思想
        • Secure By Default
          • 白名单 / 黑名单
          • 最小权限原则
        • Defense in Depth（纵深防御）
          • 在不同层面不同角度设计安全方案（木桶效应）
          • 多层过滤（有立体层次感的安全方案）
          • 在解决根本问题的地方实施针对性的安全方案（网络层 / 应用层）
        • 数据与代码分离
          • 针对漏洞：注入 / 缓冲区溢出
        • 不可预测性
          • 随机（如 id、token 等）
      • 小结
        • 安全是一种平衡的艺术
        • 寻找安全问题的本质

    • 第二章 浏览器安全

      • 同源策略——浏览器最核心最基本的安全功能

        • 即限制来自不同源的“document”或脚本对当前“document”读取或设置某些属性

        • 影响源的因素：host（根域名或 IP 地址）、子域名、端口、协议
